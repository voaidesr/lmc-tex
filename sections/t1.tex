\subsection*{Logica e baza informaticii}

Aproape toți oamenii care au fondat informatica erau logicieni. Tot ce înseamnă informatică formală se poate traduce în logică, \href{https://en.wikipedia.org/wiki/Curry%E2%80%93Howard_correspondence}{literally.}


\newpage

Asta se traduce în două lucruri:

\begin{itemize}
	\item nu o să scăpați de logică pentru încă ceva timp, în anul II aveți două materii care se construiesc pe logică și la care o să vă ajute foarte mult să aveți bazele bine stabilite.

	\item felul foarte formal de a gândi care este promovat de logică vă va ajuta foarte mult în a deveni programatori mai buni; o să ajungeți să gândiți mult mai mult ce scrieți, atât în matematică cât și în programare, pentru a fi mult mai riguros și
	      mai logic exprimat, ceea ce o să vă ajute mai ales atunci când aveți lucruri mai complicate de făcut.
\end{itemize}

Logica vă deschide drumul și către o ramură destul de nișată a informaticii, dar care este foarte bine plătită și foarte AI-proof, anume \href{https://en.wikipedia.org/wiki/Formal_verification}{verificarea formală}, care,
pe scurt, presupune demonstrarea matematică a corectitudinii unui cod, lucru care este esențial în sisteme high-risk (procesoare, sisteme medicale, rachete etc.) Facultatea noastră este foarte bogată în logicieni, se țin
foarte des seminarii de logică la \href{https://ilds.ro/}{ILDS} sau \href{https://los.cs.unibuc.ro/}{LOS}, așa că dacă vreți să faceți cercetare în timpul facultății sau să vă specializați pe partea de
verificare formală sau alte tipuri de logică, chiar aveți cu cine.

\section{Logica Propozițională}

Formulele stau la baza Logicii Propoziționale și sunt principalul lucru cu care o să lucrați capitolul ăsta. Cursul menționează că ele sunt definite inductiv:

\begin{definition}[Formule]

	Formulele sunt definite astfel:

	\begin{description}
		\item[(1)] Orice variabilă propozițională este formulă. (Adică $\forall v \in V$, $v$ este formulă)
		\item[(2)] Dacă $\varphi$ este formulă, atunci $\neg \varphi$ este formulă
		\item[(3)] Dacă $\varphi, \psi$ sunt formule, atunci $\psi \to \varphi$ este formulă
	\end{description}

\end{definition}

Dar de ce este inductiv? Până acum, ați folosit doar inducția pe numere naturale, care știți cum funcționa. Aveam un caz de bază \( P(0) \) și, presupunând că
predicatul era adevărat pentru un \( n \) natural, construiam următorul caz, \( n + 1 \), și demonstram corectitudinea.

Practic, inducția reprezintă metoda de a pleca de la ceva particular și de a generaliza incremental. Pentru numere naturale, pasul incremental este simplu, anume adunarea.

Noi putem extinde metoda asta, astfel, și la formule. Avem cazul particular, pe care îl definim a fi formulă, adică variabilele \( v \). De aici, incremental, construim
toate celelalte formule. În loc de un singur pas incremental, avem doi:

\begin{description}
	\item[\( \neg \)] putem nega formula noastră curentă pentru a obține o nouă formulă
	\item[\( \to \)] atunci când avem două formule, o nouă formulă este definită prin implicație
\end{description}

\section{Prolog Basics}
Prologul are la baza \textbf{rezolutia}, despre care o sa invatati mai tarziu in curs, dar pe scurt, se bazeaza pe a ajunge la un adevar.
Noi cand scriem cod in prolog de fapt scriem la fiecare termen \textbf{un predicat} (o functie care returneaza bool), si la chestii mai complicate
prologul practic cauta o cale catre un adevar prin predicatele pe care le definim noi.

\begin{note}[Litere mari vs mici]
	Se face distinctie intre parametrii cu litere mari si litere mici:
    
    Cele cu litere mici se refera la lucruri concrete (numere, stringuri, etc.).
    
    Cele cu litere mari sunt variabile si ele de fapt pot reprezenta orice termen in spate, fie numere, stringuri, liste etc.

    In linia de comanda cand rulam programul daca dam variabile ca parametrii prolog ii va interpreta ca necunoscute si incearca sa gaseasca valori care respecta regulile pentru acele variabile.
\end{note}

\newpage

Mai jos aveti un exemplu de niste predicate simple (fapte) si un predicat definit prin reguli:

\begin{lstlisting}[caption={Family relations in Prolog}]
% Cazurile de baza (oarecum ca la o functie recursiva)
parent(albert, bob).
parent(albert, betsy).
parent(albert, bill).

parent(alice, bob).
parent(alice, betsy).
parent(alice, bill).

% Reguli extra, prolog va incerca sa ajunga la un adevar pe care l-am definit mai sus
% Separarea prin virgula inseamna 'si' intre toate predicatele enumerate
sibling(X, Y) :-
    parent(Z, X),
    parent(Z, Y),
    X \= Y.
\end{lstlisting}

Este important sa distingem intre operatorii =, ==, 'is':
\begin{itemize}
	\item Operatorul = de fapt este operatorul de unificare, prolog va incerca sa unifice cei doi termeni cand face backtracking sa caute solutii.

	\item Operatorul == comparare stricta, nu incearca sa unifice termenii.

	\item Operatorul 'is' evalueaza expresii matematice.
\end{itemize}

Exemple:
\begin{lstlisting}[caption={Distinctie operatorii in prolog}]
?- f(X, 2) = f(1, Y).
X = 1,
Y = 2.

?- f(X, 2) == f(1, Y).
false. % Aici se face comparare fara sa incerce sa unifice termenii X si Y

?- X = 2 + 3.
X = 2+3.

?- X is 2 + 3.
X = 5.

?- 5 == 2 + 3
false. % Aici termenul '5' nu este egal cu termenul '2 + 3'
\end{lstlisting}

\subsection*{Liste in Prolog}
Listele arata vizual ca si in python: \texttt{[1, 2, abc, 4, 5]} dar modul in care lucram cu ele este diferit. In prolog ne folosim de o sintaxa cu Head si Tail:
\begin{lstlisting}[caption={Liste in prolog}]
?- [H|T] = [1, 2, 3, 4, 5].
H = 1,
T = [2, 3, 4, 5].

?- [A, B, C|T] = [1, 2, 3, 4, 5].
A = 1,
B = 2,
C = 3,
T = [4, 5].

?- [_|T] = [1, 2, 3, 4, 5].
T = [2, 3, 4, 5]. % _ este discard character, cand nu iti pasa de acel parametru
\end{lstlisting}


\textbf{Exercitiu de exemplu} (P07 din P99):

Se da o lista sub forma \texttt{[1, [2, [3, 4], 5], 6]} si trebuie sa o transformati in \texttt{[1, 2, 3, 4, 5, 6]}.

\textbf{Rezolvare:}

\begin{lstlisting}[caption={Exercitiu Flatten List}]
my_flatten([], []). % Cazul de baza

% Cazul cand elementul curent este o lista
my_flatten([H|T], R) :- 
    is_list(H),
    my_flatten(H, Fh),
    my_flatten(T, Ft),
    append(Fh, Ft, R).

% Cazul cand elementul curent nu este o lista
my_flatten([H|T], [H|Tr]) :- 
    \+ is_list(H),  % Operatorul \+ este operatorul not
    my_flatten(T, Tr).
\end{lstlisting}
